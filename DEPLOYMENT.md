# Deployment Guide — YK Academy

Step-by-step guide to deploy YK Academy to production.

---

## Prerequisites

- Node.js >= 18.17
- npm >= 9
- A [GitHub](https://github.com) account
- A [Vercel](https://vercel.com) account (free tier works)
- A [Neon](https://neon.tech) account (free tier: 0.5 GB storage)
- A Google OAuth and/or GitHub OAuth app (optional, for social login)

---

## Step 1: Set Up the Database (Neon)

1. Sign up at [neon.tech](https://neon.tech)
2. Create a new project:
   - **Project name:** `yk-academy`
   - **Region:** Choose closest to your users (e.g., AWS US East)
   - **PostgreSQL version:** 16 (default)
3. After project creation, go to the **Dashboard** and copy the **connection string**
   - It looks like: `postgresql://neondb_owner:xxxxx@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require`
4. Save this — you'll need it in Step 3

### Initialize the Schema

Option A — Locally:

```bash
# Set DATABASE_URL in .env to your Neon connection string
DATABASE_URL="postgresql://neondb_owner:xxxxx@ep-xxx.neon.tech/neondb?sslmode=require"

npx prisma generate
npx prisma db push
npx prisma db seed
```

Option B — Via Neon SQL Editor:
- Go to the Neon Dashboard → SQL Editor
- Run the migration SQL generated by `npx prisma migrate dev`

## Step 2: Set Up OAuth (Optional)

### Google OAuth

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a project or select an existing one
3. Navigate to **APIs & Services → Credentials**
4. Click **Create Credentials → OAuth 2.0 Client ID**
5. Configure:
   - **Application type:** Web application
   - **Name:** YK Academy
   - **Authorized redirect URIs:** Add both:
     - `http://localhost:3000/api/auth/callback/google` (development)
     - `https://your-domain.vercel.app/api/auth/callback/google` (production)
6. Copy the **Client ID** and **Client Secret**

### GitHub OAuth

1. Go to [github.com/settings/developers](https://github.com/settings/developers)
2. Click **New OAuth App**
3. Configure:
   - **Application name:** YK Academy
   - **Homepage URL:** `https://your-domain.vercel.app`
   - **Authorization callback URL:**
     - `http://localhost:3000/api/auth/callback/github` (development)
     - `https://your-domain.vercel.app/api/auth/callback/github` (production)
4. Copy the **Client ID** and **Client Secret**

## Step 3: Configure Environment Variables

Create a `.env` file locally and prepare all variables:

```bash
# Database
DATABASE_URL="postgresql://neondb_owner:xxxxx@ep-xxx.neon.tech/neondb?sslmode=require"

# Auth
AUTH_SECRET="your-generated-secret-here"   # Generate with: npx auth secret

# Google OAuth (optional)
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"

# GitHub OAuth (optional)
GITHUB_CLIENT_ID="your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"

# App URL
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

Generate `AUTH_SECRET`:

```bash
npx auth secret
```

## Step 5: Deploy to Vercel

1. Go to [vercel.com/new](https://vercel.com/new)
2. Click **Import Git Repository**
3. Select your `yk-academy-web` repository
4. Configure the project:
   - **Framework Preset:** Next.js (auto-detected)
   - **Root Directory:** `./` (default)
   - **Build Command:** `npx prisma generate && next build`
   - **Output Directory:** `.next` (default)
5. Expand **Environment Variables** and add all variables from Step 3:
   - `DATABASE_URL`
   - `AUTH_SECRET`
   - `GOOGLE_CLIENT_ID` (if using)
   - `GOOGLE_CLIENT_SECRET` (if using)
   - `GITHUB_CLIENT_ID` (if using)
   - `GITHUB_CLIENT_SECRET` (if using)
   - `NEXT_PUBLIC_APP_URL` → set to `https://your-domain.vercel.app`
6. Click **Deploy**

Vercel will:
- Install dependencies
- Run `npx prisma generate`
- Build the Next.js application
- Deploy to its edge network

### Build Command Notes

If you need database migrations during deployment, update the build command:

```
npx prisma generate && npx prisma migrate deploy && next build
```

## Step 6: Set Up GitHub Integration (Auto-Deployment)

Vercel automatically sets up GitHub integration when you import from GitHub:

- **Push to `main`** → Deploys to production
- **Push to any other branch** → Creates a preview deployment
- **Pull request** → Creates a unique preview URL

### Branch Strategy

| Branch | Purpose | Vercel Environment |
|--------|---------|-------------------|
| `main` | Production-ready code | Production |
| `develop` | Active development | Preview |
| `feat/*` | Feature branches | Preview (per PR) |

## Step 7: Custom Domain Setup

1. In Vercel, go to your project → **Settings → Domains**
2. Enter your custom domain (e.g., `ykacademy.com`)
3. Vercel will provide DNS records. Configure them with your domain registrar:

### Option A: Use Vercel DNS (Recommended)

Transfer nameservers to Vercel for automatic SSL and DNS management.

### Option B: External DNS

Add the following records at your domain registrar:

| Type | Name | Value |
|------|------|-------|
| A | `@` | `76.76.21.21` |
| CNAME | `www` | `cname.vercel-dns.com` |

4. Wait for DNS propagation (can take up to 48 hours, usually < 1 hour)
5. SSL certificate is automatically provisioned by Vercel

### Update Environment Variable

After the custom domain is active:

```
NEXT_PUBLIC_APP_URL="https://ykacademy.com"
```

Redeploy to apply the change.

## Step 8: Post-Deployment Checklist

### Database

- [ ] Schema is pushed to Neon (`npx prisma db push`)
- [ ] Seed data loaded (`npx prisma db seed`)
- [ ] Connection pooling enabled (Neon default)

### Application

- [ ] `AUTH_SECRET` is set
- [ ] `NEXT_PUBLIC_APP_URL` matches your production domain
- [ ] OAuth redirect URIs include production URL
- [ ] All pages load without errors
- [ ] Sign-in / sign-up flow works
- [ ] Admin dashboard is accessible with admin credentials
- [ ] Student dashboard loads correctly
- [ ] API routes return correct responses

### Optional

- [ ] Custom domain configured and SSL active
- [ ] Favicon and OG image uploaded to `public/`
- [ ] Contact form sends data correctly
- [ ] Newsletter subscription works
- [ ] File upload works (media library)

## Troubleshooting

### Build Fails with Prisma Error

**Problem:** `Error: @prisma/client did not initialize yet`

**Solution:** Ensure the build command includes `npx prisma generate`:

```
npx prisma generate && next build
```

### Database Connection Timeout

**Problem:** `Error: Can't reach database server`

**Solutions:**
1. Ensure `DATABASE_URL` includes `?sslmode=require`
2. Check Neon project is not paused (free tier auto-pauses after inactivity)
3. Verify the connection string is correct
4. Check that Neon IP allowlist is not blocking Vercel's IPs

### Authentication Redirect Loop

**Problem:** Sign-in redirects back to itself endlessly

**Solutions:**
1. Ensure `AUTH_SECRET` is set in Vercel environment variables
2. Ensure `NEXT_PUBLIC_APP_URL` matches your actual domain (no trailing slash)
3. Verify OAuth redirect URIs include the production URL

### 401 Unauthorized on Admin Routes

**Problem:** Admin API returns 401 despite being logged in

**Solutions:**
1. Verify the user's role is `ADMIN` or `SUPER_ADMIN` in the database
2. Check that the JWT contains the correct role
3. Clear browser cookies and sign in again

### Static Assets 404

**Problem:** Images or fonts return 404

**Solutions:**
1. Ensure files exist in the `public/` directory
2. Check file paths don't include `public/` prefix (e.g., use `/images/logo.png`, not `public/images/logo.png`)
3. Verify the `next.config.ts` doesn't have restrictive asset patterns

### Free Tier Database Pauses

**Problem:** First request after inactivity takes 10+ seconds

**Cause:** Neon free tier pauses the database after ~5 minutes of inactivity

**Solutions:**
1. Upgrade to Neon's Launch plan ($19/month) for always-on
2. Use a keep-alive ping (Neon provides a built-in option in project settings)
3. Accept the cold start delay for non-critical applications

### Large File Upload Fails

**Problem:** File upload times out or returns 413

**Solution:** Vercel serverless functions have a 4.5 MB body size limit by default. For larger uploads:
1. Use a presigned URL approach (S3, Cloudflare R2, or Supabase Storage)
2. Upload directly from the client to the storage provider
3. Send only the file URL to your API

## Rollback Procedure

If a deployment introduces issues:

1. Go to Vercel dashboard → **Deployments**
2. Find the last known good deployment
3. Click the three dots → **Promote to Production**

This instantly rolls back without code changes.

For database rollbacks:
1. Use Neon's **Point-in-Time Recovery** (paid plans)
2. Or restore from a manual backup:

```bash
psql $DATABASE_URL < backup_20260723.sql
```
